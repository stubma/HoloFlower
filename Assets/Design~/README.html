此前的工作

演示流程

9月底开会时与微软商定的基本演示流程为：

  1. 请小朋友在纸上画一朵花
  2. 花从纸上“长出来”，从平面图案变成三维模型
  3. 对花进行一些编辑，例如修改大小、颜色
  4. 将花放入打印机中，开始打印

指导思想是：

  1. 纯做 demo ，将所有可能存在不确定性的环节做死
  2. 尽量体现 MR 在模糊现实和虚拟世界边界的能力

基于以上思路，几个细节：

  • 花的图案是确定的，通过一些手段让小朋友照样子描一遍。
  • 花“长出来”的过程是做好的动画。
  • 在上场之前，人工做纸张位置和打印机位置的配准。这是因为 HoloLens 将所有物体一视同仁地构建为网格，无法区分哪里是打印机等。为了在正确的地方播放花“长出来”的动画和将模型放入打印机中，需要让 app 记录下纸张和打印机的位置。
  • 最好让打印机对放入其中的三维模型有遮挡。为此，做了打印机外壳的模型，不绘制而只用它做遮挡。
  • 可以考虑加入一些其它让模型看起来更“真实”的小桥段，比如太大了会被打印机卡住、丢进打印机里面的时候会晃几下之类。

程序

目前的程序仅为粗糙技术原型，Screenshot.png 是一张截图。

在 Shelf 压缩包内。（为什么叫 Shelf ：最初的构想是空中漂浮许多模型，从中选一个打印，像是一个模型架，随便取的名字。）

如何运行参考 Holograms 100: Getting Started with Unity 。

功能

  1. 移动一个托盘，记住它的位置
  2. 将一只小熊放在托盘上
  3. 对小熊进行旋转和放缩
  4. 将小熊传到局域网内的打印机上并触发打印

操作

基本操作参考 HoloLens 的官方文档和教学视频。

启动时，将看到视野正中有一个方形片，上方有一行 URL ，下方有一个 New Text 文本，右方有一只小熊。下面逐个解释。

方形片是打印机托盘的 mockup。单击它，可将它“拿起来”（此时显示环境网格），此时它将会随用户视线移动。再次单击，可放置在某个地方。程序将记住这个位置，下次进入时，仍然会出现在该处。

上方的 URL 显示了所要控制的打印机。参考脚本 DetectPrinter.cs 。

下方文本为调试日志输出。

单击右方的小熊，小熊的一个副本将会被放置在托盘上，并在周围显示四个方块和四个圆球。四个方块是放缩控件，四个圆球是旋转控件，交互方法都是单击（不放开）-拖拽（在模拟器上可能有问题，请在真机上实验）。再次点击小熊，程序会试图将小熊模型上传到打印机并触发打印。

问题说明：

  1. 现在没有处理各种异常情况，如果无法找到打印机，传文件和触发打印会导致程序异常终止。
  2. Unity 是左手系，x 向右，y 向上；打印机系统是右手系，x 向右，z 向上；因此展示的模型和传到打印机的模型实际上应该用两个不同的模型。
  3. 考虑根据可打印尺寸大小限制编辑。可打印尺寸在查找打印机时打印机对广播的回复当中有。或者考虑写死。

脚本解释

所用脚本均为 C# ，保存在 Assets/Scripts 文件夹下。下面简略说明各脚本的用处。

DetectPrinter

在局域网内发送广播寻找打印机。如果找到多个，目前取第一个。根据现场情况，可考虑改为将 IP 写死在程序中，或者仅让要演示的打印机联网。

Interaction

负责向打印机发送模型和打印指令。

LoadModel

在运行时从 assets 中加载模型。

ManipulatorUpdate

移动放缩旋转控件的总体位置，令其跟随模型。

MoveHandleManipulator

平移模型，目前没有使用。

PlaceToBed

将模型放到托盘上。

ResizeHandleManipulator

放缩控件逻辑。

ResizeHandlePosition

计算放缩控件的位置。

RotateHandleManipulator

旋转控件逻辑。

RotateHandlePosition

计算旋转控件的位置。

TargetManager

存储控件操作的对象。

交互界面设计

Design 文件夹中，附此前交互界面设计的源文件和例图。

注意这份设计受传统的平面设备影响较大，未能真正考虑在三维空间中的控件形态和交互方式，应做修改。

模型

Model 文件夹中，附打印机外壳模型（A190-KT-00 壳体模块.STL）和花朵模型动画（花 调动作_02.max）。
